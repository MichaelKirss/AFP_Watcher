Приложение позволяет осуществлять слежение за появлением файлов в определенной папке по заданной маске и копировать файлы в папку (папки) назначения.

Для слежения используется пакет java.nio.file API класса WatchService. Watch Service API на низкоуровневом уровне следит за событиями 
происходящими в файловой системе. В приложении используется подписка на событие API создания файла "ENTRY_CREATE".

Настройка приложения. Все необходимые настройки выведены в конфигурационный файл config.properties. 
source.dir - каталог за которым осуществляется слежение
target.dir - каталог в который копируются файлы
ftc.dir - каталог в который копируются файлы для переноса в систему приемник
file.regex - регулярное выражение для отбора файлов по заданной маске.

Файл настроек в проекте сформирован под конкретную задачу.

Логгирование в приложении осуществляется с использованием пакета org.sl4j.LoggerFactory
пример конфигурационного файла логгирования приложен в примере /resources/logback.xml

Сборка проекта осуществляется в Maven. 
Пример конфигурационного файла для сборки приложения Maven в продуктивную среду pom.xml 
в командной строке терминала 
mvn clean package
